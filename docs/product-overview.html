    participant O as orchestrator
    participant A as adapters
    participant C as compile (Two-Step CoT)

    participant Q as question
    participant S as stance

    participant GL as graph_layers
    participant DR as triggers

    participant PDF as pdf_utils

    U->>O: ingest_article(vault, article)
    O->>A: fetch + format normalization
    A-->>O: Markdown + metadata
    O->>O: write raw/articles/{slug}.md
    O->>C: try_llm_compile()
    Note over C: Step 1: fact_extraction
Step 2: structure_compile (constrained by ingest_compile_v2)
    C-->>O: compiled_payload (v2 JSON)
    O->>P: build_brief_page()
    P-->>O: brief md
    O->>PDF: brief_to_pdf()
    PDF-->>O: {slug}.pdf
    O->>P: build_source_page()
    O->>T: ensure_taxonomy_pages()
    T->>T: concepts/ + entities/ + domains/
    O->>Q: write_question_page()
    O->>S: apply_stance_impact()
    O->>T: ensure_comparison_page()
    O->>G: write_knowledge_graph()
    O->>GL: build_all_domain_subgraphs()
    Note over O: claim evolution 由 LLM 按 claim_evolution.md 分析
    O->>R: collect_ingest_data() → LLM 按 ingest_impact.md 分析
    O->>DR: detect_triggers()
    DR-->>O: deep research suggestions
    O-->>U: result (status + PDF + report + triggers)

Deep Research Flow
深度研究完整时序
sequenceDiagram
    participant U as 用户
    participant A as Claude Code (LLM)
    participant DR as deep_research.py
    participant DL as dependency_ledger
    participant V as validation
    participant PDF as pdf_utils

    U->>A: "深入研究 X"
    A->>A: Phase 0: 读 hot.md + 立场 + 问题
    A->>U: Phase 1: 为什么重要？
    U-->>A: 真实需求
    A->>A: Phase 2: 形成 2-4 假说（按 research_hypothesis.md 约束）
    A->>DR: init(topic, hypotheses)
    DR->>DL: init_ledger_page()
    DR-->>A: ledger_path
    A->>DR: Phase 3: collect_vault_evidence()
    DR-->>A: candidates（脚本预排序，LLM 按 F/I/A 重新分类）
    loop Phase 4: Adaptive Rounds
        A->>A: WebSearch 确认 + 反驳
        A->>A: 结果走 ingest pipeline
        A->>DL: update-confidence (+/-15-35%)
        A->>DR: check-sufficiency()
        alt 不通过
            A->>A: 继续下一轮
        end
    end
    A->>A: Phase 5: 外部事实校准
    A->>A: Phase 6: 根本问题挖掘
    A->>DR: Phase 7: record-scenarios()
    A->>DR: Phase 8: record-premortem()
    A->>A: Phase 9: 打包 Why/What/How/Trace
    A->>DR: finalize_report()
    Note over DR,V: Phase 9.5: 质量门控
    DR->>V: validate_report()
    V->>V: 7 项红线 + 依赖链审查
    V-->>DR: ValidationResult
    DR->>DR: 附录 A 写入报告
    DR->>PDF: report_to_pdf()
    PDF-->>DR: {slug}--report.pdf
    DR-->>A: {report_path, pdf_path, validation}
    A->>DR: update_closure()
    DR->>DR: 更新 question/stance/hot/log
    A-->>U: 报告 + PDF + 验证结果
