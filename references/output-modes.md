# 输出格式参考

查询时 LLM 根据用户意图选择输出格式。完整行为指南见 `references/query-guide.md`。

## 格式速查

| 用户意图 | 输出格式 | 说明 |
|---------|---------|------|
| "什么是 X"、"总结" | 快速了解 | 3-5 句 + 来源引用 |
| "深入分析"、"综述" | 深度综合 | 多来源交叉的结构化报告 |
| "对比 X 和 Y"、"vs" | 对比分析 | 对比表 + 关键差异 |
| "演变"、"时间线" | 时间线 | Mermaid gantt + 事件说明 |
| "准备开会"、"汇报" | 认知简报 | 要点 + 反面 + 待讨论问题 |
| "反驳"、"质疑" | 反驳材料 | 最强反面证据 + 冲突信号 |
| "写文章"、"帮我写" | 文章草稿 | 论点 + 证据 + 叙述结构 |
| "学习路径"、"系统学习" | 学习路径 | 按依赖排序的推荐阅读 |
| "素材"、"给 LLM" | 素材包 | 带编号来源摘要 + 原文片段 |

## 设计原则

- **LLM 主控**：格式选择由 LLM 基于意图理解决定，不依赖正则匹配
- **口语化**：用户不需要知道格式名称，只需要说自然语言
- **组合需求**：拆成多次查询分别处理（如"对比 X 和 Y，然后写文章"）
- **模糊意图**：默认走"快速了解"，回答后追问是否需要更深入的形式

## 输出位置

查询结果默认写入 `wiki/outputs/`（`lifecycle: temporary`）。用户确认后可通过 `/note` 流程沉淀为正式页面。
